## データ構成

ページを構成する際のデータ構成を以下に示します。Block要素を明確に分類するのが特徴です。

| 名前       | 説明                                                                                                                            |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------- |
| Pages      | 各ページのデータです。Astroの標準機能を使用します。<br/>/src/pages/                                                             |
| Layouts    | プロジェクト全体で使用する共通レイアウトです。すべてのページで使用します。<br/>/src/layouts/                                    |
| Blocks     | 各ページを構成する要素です。積み上げます。<br/>/src/blocks/                                                                     |
| Components | LayoutsやBlocks内で使用する再利用可能なパーツです。componentにはmarginは設定せず、配置する側で設定します。<br/>/src/components/ |
| Data       | サイトに必要なデータをDataで管理します。<br/>/src/data/                                                                         |

```plaintext
/
├── dist      ...buildしたデータ
├── public    ...画像データ（public画像）などの静的アセッツ
├── scripts   ...開発に必要なnode.jsのコード（必要に応じて）
├── src       ...開発データ
│ ├── blocks      ...ブロックデータ
│ ├── components  ...部品データ
│ ├── data        ...案件に関わるデータ
│ ├── icons       ...アイコン用のsvgデータ
│ ├── image       ...画像データ（src画像）
│ ├── layouts     ...各ページで使用するレイアウト
│ ├── pages       ...各ページのデータ
│ ├── script      ...コンポーネントに依存しないスクリプト
│ ├── stories     ...Storybook用のデータ
│ ├── styles      ...共通スタイル
│ └── types       ...型データ（必要に応じて）
└── tokens    ...デザイントークンファイル
```
